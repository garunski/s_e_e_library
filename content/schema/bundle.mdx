# Bundle schema

Category `bundle`. No payload of its own.

A bundle's catalog `files[]` array installs the payloads of several other
packages together; the build infers each file's kind from its `to` prefix (see
[Install paths](/schema/#install-paths)).

Bundle metadata lives in its `s_e_e_package.json`:

- `id` - bundle id (e.g. `bundle-author-story`).
- `slug` - url/file-safe slug.
- `category` - `bundle`.
- `name` - display name.
- `description` - one-line summary.
- `labels` - string array.
- `dependencies` - string array of the package `id`s it pulls in.

## JSON Schema

Schema for a bundle's `s_e_e_package.json`. Download:
[bundle-package.schema.json](https://garunski.github.io/s_e_e_library/schema/bundle-package.schema.json)

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://garunski.github.io/s_e_e_library/schema/bundle-package.schema.json",
  "title": "Bundle metadata (s_e_e_package.json)",
  "type": "object",
  "required": ["id", "slug", "category", "name"],
  "properties": {
    "id": { "type": "string", "minLength": 1 },
    "slug": { "type": "string", "minLength": 1 },
    "category": { "const": "bundle" },
    "name": { "type": "string", "minLength": 1 },
    "description": { "type": "string" },
    "labels": { "type": "array", "items": { "type": "string", "minLength": 1 } },
    "dependencies": { "type": "array", "items": { "type": "string", "minLength": 1 } }
  }
}
```

The bundle's `files[]` entries follow the `file` definition in the
[catalog schema](/schema/#json-schema).

See the [bundle authoring guide](/authoring/bundles) and the
[catalog schema](/schema/).
